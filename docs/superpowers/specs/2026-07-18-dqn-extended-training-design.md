# DQN PoC extended training — design

Date: 2026-07-18
Status: approved (user), this session
Context: follows task 15 (`docs/superpowers/specs/2026-07-17-task15-dqn-poc-design.md`).
The 800-episode result was re-run this session and reproduced byte-identically
(DP 0.636 > DQN 0.581 > greedy 0.533 > random 0.197).

## Goal

Train the DQN 5× longer (800 → 4000 episodes) and, if it scores at least as well
as the published 0.581 mean return, make the 4000-episode run the headline PoC
result everywhere it is published.

## Decisions (user-approved)

1. **Episodes**: `EPISODES = 4000` in `ml/research/dqn_poc.py`. Nothing else about
   the protocol changes — seeds (env 0/123, agent 0), eval (100 episodes, seed 123),
   DP/greedy/random baselines, and the epsilon schedule
   `max(0.05, 1.0 - ep / 400)` all stay as-is. Consequence: the first 800 training
   episodes exactly replicate the published run; the new learning curve is a strict
   extension of the old one.
2. **Results placement**: replace the main result. Regenerated
   `ml/reports/dqn_poc.{png,md}` become the committed artifacts; the measured table
   and any prose citing the DQN number in `docs/learning-path-optimization.md` §6.1
   and `ml/reports/EVIDENCE.md` are updated. Live re-runs at defense then take
   ~8 min instead of ~2.5.
3. **Decision rule**: replace only if the 4000-episode greedy eval mean return
   ≥ 0.581. If it comes back worse (late-training drift on the rolling 10k buffer
   is possible), commit nothing, try the fallback schedule
   `max(0.05, 1.0 - ep / (EPISODES // 2))` once, and if that also fails to beat
   0.581, keep the published result and document the negative finding in the
   ledger instead.
4. **Report staleness cleanup** (targeted, in scope): the notes section of the
   generated `dqn_poc.md` currently hardcodes the measured gaps
   ("DQN beats greedy by +0.048 (+9%)…"). Compute that sentence from the results
   dict so re-runs can never leave stale numbers in the report. No other report
   text changes.
